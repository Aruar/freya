Frequent Asked Questions


Q: I could not download the svn because of a strange file. I'm under linux Debian.
A: Type: LANG=en_EN.UFT8 svn co 'url of SVN repository'. You must be in UTF8.

Q: How can I compile easily Freya under Windows?
A: Go to http://www.cygwin.com.
Click on the 'Setup' link, and choose 'Open'.
You should get a screen with version of cygwin. Click 'Next'.
Select 'Install from Internet', then click 'Next'.
Choose where you want to install it (C:\cygwin\ is default). Make sure 'Install for All Users' and 'Unix Install Type'/'Unix/Binaries' are selected. Then click 'Next'.
Choose where you want the temporary files to be stored. Click 'Next'.
Select how you connect to the Internet. Click 'Next'.
Choose the mirror you want. The one I have selected ran around 70~80kb/s for me. Click 'Next'.
To default choices, add at least these packages:
- subversion: A version control system
- gcc: C compiler upgrade helper
- gcc-core: C compiler
- gcc-g++: C++ compiler
- gcc-mingw-core: Mingw32 support headers and libraries for GCC
- gcc-mingw-g++: Mingw32 support headers and libraries for GCC c++
- libiconv: GNU character set conversion library and utilities'
- make: The GNU version for the 'make' utility
- mingw-runtime: MinGW Runtime
- mktemp: allows safe temp file/dir creation from shell scripts
- gcc-g++: C++ compiler
Then click 'Next'. You should now start downloading and installing cygwin.
When it's down installing, click 'Finish'.
Now a small popup comes up. Click 'OK'.
Delete the temporary folder it created.
Open cygwin now. You should see a screen with a prompt.
Type ' cd C: '.
Type ' cd where\your\Freya\is '.
Now type ' make clean '.
Finally type ' make '.
And you should get no error!
YOU ARE NOW FINISHED!
To play, type ' ./start ' or ' ./athena-start start '.

Q: When compiling under cygwin, I have the error message "'file_name':'line_number': internal compiler error: segmentation fault."
A: Modify the 'Makefile' file of the main directory in order to take correctly into account your configuration (computer).
Now Type ' make clean '.
Then Type ' make '.
And you should get no error!

Q: I get the error message "This application has failed to start because cygwin1.dll was not found. Re-installing the application may fix this problem."
A: You're missing the cygwin dlls. Please get the latest dll at cygwin web site: http://www.cygwin.com. If you're unsure, asking around you, but always remember - use common sense and search before asking.

Q: My map-server can not load maps.
A: Edit your conf/grf-files.txt file and verify if all your grf files have correct path.
Note: only one grf file is needed, if all mentioned maps are inside.
If paths are good, check if you have all maps in grf files. Then, remove missing maps from the list inside conf/map_athena.conf file.

Q: My server starts and stops immediatly (windows/win32 compilation).
A: You probably don't have installed the 'zlib.dll' library on your computer.
Download this file on Internet, and copy it in the server's directory or in the 'system32' directory of the windows directory.

Q: How can I create rapidly an account for my tests?
A: In the 'login_athena.conf' file, activate the 'new_account' option.
Start Freya.
You connect, and type an accout name followed by _ F or _ M according to the sex which you wish.
Type a password with a minimum of 4 characters, followed by '@' and the administration password setted in the 'login_athena.conf' file ('admin_pass' option).
While validating, you will obtain 'unknown account'. However, your account will have been created.
Type the account name (without _ F or _ M) and the password (without the '@' followed by the administration password).
Validate and play.

Q: How can I create an account for a player?
A: Start Freya.
Run the PERL software 'ladmin' from the directory /bin/tool.
If it can not connect to your login-server, check the settings of connection at start of the file 'ladmin' with a text editor.
Type 'help', to display commands (including the one to create accounts).

Q: How can my players create an account to play?
A: Start Freya.
Tell to your players to connect with a web explorer on http://:.
Now, they can create their account on the displayed page.
Note: you must have activate the httpd server (enabled by default).

Q: I connect login-server, then choose a character, press enter and... !immediately! see message "disconnected from server".
A: It's the packet database related problem. Edit your packet_db.txt file from db/ directory. It looks like:
//2005-05-09aSakexe by Sara
0x009b,26,wanttoconnection,4:9:17:18:25
0x0089,8,ticksend,4
...
//2005-06-28aSakexe
0x009b,32,wanttoconnection,9:15:23:27:31
0x0089,13,ticksend,9
... And so on...
Now if your exec is from (for example) 2005-05-26, comment (//) all that's above 2005-05-09 (but don't comment first segment - shared packets) and comment all that's below the next packet versions (from 2005-06-28 in this case).
Uncomment 2005-05-26 section.
Then, restart map-server and play.

Q: What is the 'import' file for?
A: Most people don't know the real use of the 'import' files. After you do, you will wonder what you ever did without it.
The main thing it does, is provide a way for you to change your config settings without having to modify the actual files. You can test some settings by this way.

Q: How does the 'import' file work?
A: You place only the settings you have changed in the 'import' files.
Say you want to change your job experience rate from the default (100) to 2x (200). Well then you would place this in your import/battle_conf.txt file (folder and name of the 'import' file are mentioned in your 'import' line):
job_exp_rate: 200
You don't need the comment (it's only a comment!), but you can add it for clarity sake.
Now this new setting takes place over the setting in battle_athena.conf file.
How? Simple. When the server reads the 'import' line, it reads the mentioned file and all values inside this file replace previously readed values. The last readed setting of each option is the used setting.
But, if you add another 'import' file with this option or change this option after the 'import' line, your 'import' setting will not be used. The last readed value is always the value used by the server.
